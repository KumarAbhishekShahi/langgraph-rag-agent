REST API Design Standards

URL Conventions

  • Use nouns not verbs: /api/v1/orders not /api/v1/getOrders
  • Plural resource names: /users, /products
  • Lowercase hyphen-separated: /order-items
  • Version in URL: /api/v1/

HTTP Methods

MethodUseIdempotent
GETRead resourceYes
POSTCreate resourceNo
PUTFull updateYes
PATCHPartial updateNo
DELETERemove resourceYes

Standard Response Structure

{
  "success": true,
  "data": { },
  "message": "Order created successfully",
  "timestamp": "2026-03-31T10:15:00Z",
  "requestId": "req-abc-123"
}

Error Response

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Email is required",
    "field": "email"
  }
}

Pagination

All list endpoints must support cursor-based pagination:

GET /api/v1/orders?page=2&size=20&sort=createdAt,desc

Rate Limiting

  • Default: 100 requests/minute per API key
  • Burst: 200 requests/minute for authenticated users
  • Return 429 Too Many Requests with Retry-After header
